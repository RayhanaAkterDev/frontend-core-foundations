Mobile-first vs Desktop-first Approach

In responsive web design, the way you write and prioritize your CSS can follow two strategies: mobile-first or desktop-first. Understanding the difference helps you structure your styles efficiently.

üì± Mobile-first Approach

Mobile-first design starts by targeting small screens (like phones) with base styles. You then progressively enhance the layout using min-width media queries for larger devices. 


/* Base styles for mobile devices */
.container {
  font-size: 16px;
}

/* Styles for tablets and up */
@media (min-width: 768px) {
  .container {
    font-size: 18px;
  }
}

Desktop-first design begins with styles for larger screens, then uses max-width media queries to scale down for smaller devices. 


/* Base styles for desktop devices */
.container {
  font-size: 18px;
}

/* Styles for tablets and below */
@media (max-width: 768px) {
  .container {
    font-size: 16px;
  }
}

Modern best practice recommends the mobile-first approach:

  • Aligns with progressive enhancement principles
  • Delivers faster performance on mobile devices
  • Fits well with the natural mobile user base growth
