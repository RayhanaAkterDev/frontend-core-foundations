Introduction to CSS Preprocessors: Sass & LESS

CSS preprocessors extend the capabilities of plain CSS by adding features like variables, nesting, mixins, and functions. Two popular preprocessors are Sass and LESS.

What is Sass?

Sass (Syntactically Awesome Stylesheets) uses the .scss or .sass file extension and compiles into regular CSS. Itâ€™s widely used and offers powerful features.

Sass Features:

  • Variables: Store values like colors or sizes.
  • Nesting: Organize styles inside selectors.
  • Mixins: Reuse chunks of CSS code.
  • Partials & Imports: Split styles into manageable files.

Example (SCSS syntax):

$primary-color: #3498db;

nav {
  background: $primary-color;
  ul {
    list-style: none;
    li {
      display: inline-block;
      a {
        text-decoration: none;
        color: white;
      }
    }
  }
}

What is LESS?

LESS is another preprocessor that uses the .less extension. It is similar to Sass and also compiles into CSS.

LESS Features:

  • Variables: Declared using @ symbol.
  • Nesting: Just like Sass.
  • Mixins: Declare reusable rules.
  • Operations: Do math inside stylesheets.

Example (LESS syntax):

@main-color: #27ae60;

header {
  background: @main-color;
  nav {
    a {
      color: white;
      padding: 10px;
    }
  }
}
